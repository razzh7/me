## 为什么

在我初学前端那会，我的博客园博客中引入了 aplayer，当时就觉得这个播放器很好用，留下了印象，在去年年初做博客重构的时候，我也写了一篇文章说过要在自己的博客中引入 `aplayer`，然后就拖到了今年😆。  

但是，调研了一下 aplayer，它并不支持在 `Dark` 模式下的暗黑主题，我的博客默认的主题风格就是 `Dark` 风格，我希望播放器能够自适应适配我博客的主题，所以让我萌生出了自己造一个 `aplayer` 的想法。 

## 特性
- 支持 React 19
- 移动端适配-迷你模式
- 支持歌词滚动显示
- 暗黑和明亮主题切换
- 通过 CSS 变量自定义样式
- 支持 SSR
- 友好的可访问性

此外，aplayer 中使用的 icon 图标都来自我自己写的库： [twist-icons](https://github.com/twist-space/twist-icons) 😂，就是想多用用自己写的库哈哈哈。
